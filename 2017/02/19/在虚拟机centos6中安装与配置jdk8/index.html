在虚拟机centos6中安装与配置jdk8

发表于 2017-02-19   |   分类于 操作   |  

环境简述：通过xshell连接虚拟机，在xshell上进行的操作，采取的安装方式是rpm，安装软件：jdk-8u25-linux-x64.rpm
参考

jdk官网下载地址
下面根据我的实际操作展开：
首先检查自己系统里面是否默认安装了openJDK,如果安装了需要先卸载掉，之后开始下载jdk。我的虚拟机下载的是centos mini，所以没有自带
用windows10系统里的浏览器打开软件下载网页后，要记得选中Accept License Agreement才能下载
刚开始我的做法是右击下载链接，选择复制链接然后直接在xshell中输入wget http://download.ora… /jdk-8u121-linux-x64.tar.gz
有没有发现这样做下载得很快？快就对了！因为这样一来它没有下全，等下软件也是不可用的
为次，我的做法是先在windows10下点击链接直接下载，下载完后再通过rz传到centos系统
如果你下载的是.tar.gz包，解压缩的时候$ tar -zxvf *.tar.gz可能会遇到问题，这时有两种解决办法：

  1. $ tar -xvf *.tar.gz 原因是这个压缩包没有用gzip格式压缩，所以不用加z指令
  2. 把下载下来的文件重命名为.tar再tar –zxvf .tar
    如果下载的是rpm,则把windows下载好的rpm包通过xshell传到/usr/local/，接着执行以下操作:

    [root@zck local]# rpm -ivh jdk-8u25-linux-x64.rpm

接着等待安装就好了，JDK默认安装在/usr/java中
执行以下操作，查看信息是否正常：

[root@localhost ~]# java
[root@localhost ~]# javac
[root@localhost ~]# java -version
java version “1.8.0_25”
Java(TM) SE Runtime Environment (build 1.8.0_25-b17)
Java HotSpot(TM) 64-Bit Server VM (build 25.25-b02, mixed mode)

配置环境变量
虽然我的机器安装完jdk-7-linux-x64.rpm后不用配置环境变量也可以正常执行javac、java –version操作，但后面因为要配置tomcat，Eclipse/NetBeans/Tomcat等软件就是通过搜索JAVA_HOME变量来找到并使用安装好的jdk，所以还是要进行配置，否则tomcat找不到jdk，尽管它们放在同一个文件下，操作如下：
修改系统环境变量文件

vi + /etc/profile

向文件里面追加以下内容：

JAVA_HOME=/usr/java/jdk1.8.0_25
JRE_HOME=/usr/java/jdk1.8.0_25/jre
PATH=$PATH:$JAVA_HOME/bin:$JRE_HOME/bin
CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar:$JRE_HOME/lib
export JAVA_HOME JRE_HOME PATH CLASSPATH

使修改生效

[root@localhost ~]# source /etc/profile //使修改立即生效
[root@localhost ~]# echo $PATH //查看PATH值

查看系统环境状态

[root@localhost ~]# echo $PATH
/usr/local/sbin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin:/root/bin:/usr/java/jdk1.8.0_25/bin:/usr/java/jdk1.8.0_25/jre/bin
