我理解的计算机的通信过程

发表于 2017-02-19   |   分类于 总结   |  

个人对于计算机网络通信过程的理解，不足之处望指正

【情景创设】：假设你家里有两台电脑，DHCP协议给这两台电脑自动分配IP地址，电脑A分到的IP为192.168.2.2/24，电脑B分到的IP为192.168.2.3/24，然后这两个电脑都是通过家里的同一个路由器访问外网，该路器有两个端口：WAN 口和 LAN口。WAN：接外部 IP 地址用，转发来自内部 LAN 接口的 IP 数据包，这个口的 IP 是唯一的。LAN：接内部 IP 地址用

##【情景一】A电脑向B电脑发送数据(相同局域网内的通信)
1.在A电脑中，进行数据的封装之前先检查所需要的目的物理地址等信息是否已知，如果未知，就向本网段的所有主机发送ARP数据包，该数据包包括的内容有：源主机IP地址，源主机MAC地址，目的主机的IP地址。当本网络的所有主机收到该ARP数据包时，首先检查数据包中的IP地址是否是自己的IP地址，如果不是，则忽略该数据包，如果是，则首先从数据包中取出源主机的IP和MAC地址写入到ARP列表中，如果已经存在，则覆盖，然后将自己的MAC地址写入ARP响应包中，告诉源主机自己是它想要找的MAC地址。源主机收到ARP响应包后。将目的主机的IP和MAC地址写入ARP列表

2.准备好后，应用层产生数据交给传输层；传输层进行数据的分段，使数据的大小适合在网络上传递，根据应用层不同应用产生的数据，选择不同协议栈进行封装TCP或UDP,再进行不同服务端口号的封装，再传递给网络层；网络层在数据包头部进行封装源IP、目的IP，根据上层协议栈标识不同协议号，当IP包头封装TCP时协议号为6、UDP协议号为17，用来标识上层使用的协议，继续传递给数据链路层；数据链路层在数据包头部进行帧的封装，封装源MAC、目的MAC。接着交付给物理层，物理层转换成电信号（bit流）

3.B电脑接受到信息，便对数据进行解封装，发现目的ip是给自己的，然后继续解封装，最后把数据传送给相应的应用程序。通信完成

##【情景二】A电脑访问百度网页 (局域网与外网的通信)
1.客户端浏览器通过DNS解析到www.baidu.com的IP地址。其余的1，2步骤相同

3.数据包传到路由器的LAN接口，路由器对数据进行解封装，接着扫描路由表的每一行，用里面的子网掩码与数据包中的目标IP地址做逻辑与运算（&）找出目标网络号，发现目的IP不在本局域网，则发给默认路由WAN口，如果有开启NAT服务，且源IP地址为NAT设置的源IP地址，于是进行NAT转化，（例如：192.168.1.2:80 -> 10.221.0.24:8080,所以，从 WAN口出去的地址为：10.221.0.24:8080
通过这样的层层端口映射，最终保证地址(IP + 端口)的唯一性），接着查看下一跳路由，再接着传到数据链路层，把源MAC地址改为该路由的MAC地址，目的MAC改为下一跳路由的MAC地址。封装后交到物理层进行传输

4.在接下来各结点的传输过程中，源IP和目的IP都没有变，但MAC地址在不断的变化，目的MAC都是通过：路由器查找路由表得出下一跳路由IP,再根据ARP缓存表得到目的MAC，找不到则进行ARP广播。直到百度的服务器收到数据包，接着对数据包层层解封装，最后读取内容

5.当百度服务器返回数据时，也是类似上面的过程在各节点间转发，直到转发到本地路由的WAN口时，再进行NAT，得到A电脑的私有IP，最终发给A电脑
