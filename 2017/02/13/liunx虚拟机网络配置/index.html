liunx虚拟机网络配置

发表于 2017-02-13   |   分类于 linux   |  

在虚拟机上配置刚安装的centos6.3系统的网络

1.修改配置文件

(1)/etc/sysconfig/network-scripts/ifcfg-eth0 网卡配置文件的内容

DEVICE=eth0 #网卡设备名称，此处的名称与文件名中的名字要相同
HWADDR=00:23:54:68:84:DA #MAC地址
TYPE=Ethernet #类型，类型为以太网
UUID=674a7aa6-8829-431b-84c8-b2daf6cfa7bc #唯一识别码用于识别是否是同一台计算机，若一个局域网内存在多台同样UUID的计算机将无法正常上网
ONBOOT=yes #是否随网络服务启动，eth0生效
NM_CONTROLLED=yes #是否可以由Network Manager 图形管理工具托管
BOOTPROTO=static #是否自动获取IP(none、static、dhcp)
IPADDR=192.168.0.37 #IP地址
NETMASK=255.255.255.0 #子网掩码
GATEWAY=192.168.0.1 #网关
DNS=192.168.0.1 #DNS
IPV6INIT=no #IPv6是否启用
USERCTL=no #不允许非root用户控制此网卡

(2)主机名修改文件/etc/sysconfig/network

NETWORKING=yes #注意看这一行是不是yes，默认是yes
HOSTNAME=localhost.localdomain

(3)DNS配置文件/etc/resolv.conf
resolv.conf的关键字主要有四个，分别是：

  • nameserver //最主要关键字,定义DNS服务器的IP地址,可定义多个，且只有当第一个nameserver没有反应时才查询下面的nameserver
  • domain //定义本地域名，很多程序用到它，如邮件系统；当为没有域名的主机进行DNS查询时，也要用到。如果没有域名，主机名将被使用，删除所有在第一个点( .)前面的内容
  • search //定义域名的搜索列表,当要查询没有域名的主机，主机将在由search声明的域中分别查找,domain和search不能共存；如果同时存在，后面出现的将会被使用
  • sortlist //对返回的域名进行排序
    修改完文件记得service network restart重启网络服务

    2.修改UUID(克隆机才需要修改)

    1)删除/etc/sysconfig/network-scripts/ifcfg-eth0中的MAC地址行
    2)rm -rf /etc/udev/rules.d/70-persistent-net.rules删除网卡和MAC地址绑定文件
    3)重启系统

    3.设置虚拟机网络连接方式

    虚拟机选择网络设置三种方式
  • 桥接：相当于虚拟机与真实机利用真实的网卡进行通信，虚拟机与真实机在同一网段时可以使用，占用真实网段的一个ip地址，桥接可以和局域网通信也可以访问外网
  • NAT模式：真实机与虚拟机通信利用虚拟网卡，nat能访问外网，但不能和局域网内的计算机通信
  • 仅主机模式:真实机与虚拟机通信利用虚拟网卡，仅主机模式指只能和自己本机通信
    在选择桥接模式之后，需要选择桥接的网卡（即需要确定虚拟机需要通过哪个真实网卡连接网络），一般的虚拟机都是自动桥接，但是这种方式不靠谱，所以你需要自己选择，如果你是通过有线接入网络，则选择有线桥接，若是通过无线接入网络，则通过无线接入。
