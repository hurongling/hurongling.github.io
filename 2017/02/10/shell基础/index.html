shell基础

发表于 2017-02-10   |   分类于 基础   |  

linux基础知识，关于shell的一些简单命令及快捷键，如历史命令，重定向，管道符

echo输出命令

echo [选项] [输出内容]

-e 支持反斜线控制的字符转换
\e[1;nm 开启颜色显示（n取值30-37）
\e[0m 取消颜色显示
echo -e “\e[1;31m 我爱学习 \e[0m “

Bash脚步

  1. 以#!/bin/bash开头
  2. 脚本执行：bash hello.sh 或 赋予执行权限，直接运行chmod 755 hello.sh接着./hello.sh

    别名

    alias #查看系统中所有命令的别名
    alias 别名=’原命令’ #设定命令别名（临时）
    vi ~/.bashrc #编辑文件永久修改别名
    unalias 别名 #删除别名

命令生效顺序

  1. 绝对路径或相对路径执行的命令
  2. 执行别名
  3. 执行Bash内部命令
  4. 执行按照$PATH环境变量定义的目录查找顺序找到的第一个命令

    常用快捷键

  • ctrl+c 强制终止当前命令
  • ctrl+l 清屏
  • ctrl+a 光标移动到命令行首
  • ctrl+e 光标移动到命令行尾
  • ctrl+u 从光标所在位置删除到行首
  • ctrl+z 把命令放入后台
  • ctrl+r 在历史命令中搜索

    历史命令

    history [选项][历史命令保存文件]

    -c 清空历史命令
    -w 把缓存中的历史命令写入~/.bash_history
    上下箭头调用以前的历史命令
    !n重复执行第n条历史命令
    !!重复执行上一条命令
    !字串 重复执行最后一条以该字串开头的命令
    历史命令默认保存1000条，可在/etc/profile中修改HISTSIZE

输出重定向

  • 命令 > 文件 #以覆盖方式将正确输出结果重定向到指定文件
  • 命令 >> 文件 #以追加方式将正确输出结果重定向到指定文件
  • 错误命令 2> 文件 #以覆盖方式将错误输出结果重定向到指定文件
  • 错误命令 2>> 文件 #以追加方式将错误输出结果重定向到指定文件
  • 命令 > 文件 2>&1 #以覆盖方式将正确和错误输出结果保存到同一文件
  • 命令 >> 文件 2>>&1 #以追加方式将正确和错误输出结果保存到同一文件
  • 命令 &> 文件 #同 命令 > 文件 2>&1
  • 命令 &>> 文件 #同 命令 >> 文件 2>>&1
  • 命令 >> 文件1 2>>文件2 #把正确输出追加到文件1中，把错误输出追加到文件2中

管道符

命令1 | 命令2

  • 命令1的正确输出作为命令2的操作对象
  • 命令2出错则命令2不执行
  • ll -a /etc/ | more
  • netstat -an | grep “ESTABLISHED”

    Bash中特殊符号

  • ‘’ 单引号中所有的特殊符号，如$,反引号都没有特殊含义
  • “” 双引号中特殊符号没特殊含义，但$,\,反引号除外
  • ` 反引号括起来的命令在BASH中会先执行
  • $() 和反引号效果一样

  • 在shell脚本中，#代表注释

  • $ 调用变量的值，如$name
  • \ 转义符
