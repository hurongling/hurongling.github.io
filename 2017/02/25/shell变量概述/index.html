shell变量概述

发表于 2017-02-25   |   分类于 基础   |  

介绍shell变量的分类及基本操作

一、shell与其他语言的区别

  1. 不管是PHP ，还是JAVA ，它是用来编写应用程序的，或是网站； JAVA主要是实现服务端程序。而shel编程，它是一个脚本语言（所见即所得）。它不需要执行编译过程之后再执行；它是编译过程放在执行过程中，所以，执行起来要慢得多
  2. SHELL，主要是帮助管理员，简化管理操作。比如 “批量生成文件、新用户，定时备份文件，定时检测系统等”

二、shell变量的分类

1.用户自定义变量
为局部变量
2.环境变量
这种变量中主要保存的是和系统操作环境相关的数据，变量可以自定义，但是对系统生效的环境变量名和变量作用是固定的，为全局变量
3.位置参数变量
这种变量主要是用来向脚本当中传递参数或数据的，变量名不能自定义，变量作用是固定的
4.预定义变量

#是Bash中一景定义好的变量，变量名不能自定义，变量作用也是固定的
注意：

  • 在Bash中，变量的默认类型都是字符串类型。$a=1,$b=2,$c=$a+$b,echo $c，会输出1+2
    解决方法：
  1. c=$(($a+$b))
  2. c=$[$a+$b]
  3. c=$(expr $a + $b) #注意+号左右要有空格
    这样$c才等于3
  • 变量等号两侧不能加空格，若变量值中间有空格，必须用双引号括起来
  • 变量名外面的花括号是可选的，加不加都行，加花括号是为了帮助解释器识别变量的边界，比如下面这种情况：

    skill=java
    echo “I am good at ${skill}Script”

三、设置环境变量

export 变量名=变量值
或
变量名=变量值
Export 变量名

四、变量叠加的两种方法：

x=123
x="$x"456
x=${x}789
echo $x 为123456789
实用例子：PATH=”$PATH”:/root/sh

##五、变量的查看：
set #查看系统中所有已经定义的变量
env #查看环境变量
set -u 输入后再调用未声明变量时会报错，用来区分是未声明变量还是值为空的变量。例如：
set -u
echo $a

##六、变量的删除：
unset 变量名，而不是$变量名

##七、语系变量
locale -a | more #查看Linux支持的所有语系环境
echo $LANG #当前语系环境
echo $LC_ALL #整体语系环境

八、位置参数变量

$n n为数字，$0代表命令本身，$1-$9代表第一到第九个参数，10以上的参数需要用大括号包含，如${10}
$ 代表命令行中的所有参数，$把所有参数看成了一个整体
$@ 也代表命令行中的所有参数，$@把每个参数区分对待
$# 代表命令行中所有参数的个数
$*与$@的区别

for i in “$*”
do
echo “The parameters is :$i”
done

for循环只会循环一次，如果改为for i in “$@”则会循环$#次

九、预定义变量

$? 如果这个命令返回值为0，说明上个命令正确执行，如果非0说明上个命令执行不正确
$$ 返回当前进程PID号
$! 返回后台运行的最后一个进程的PID号

十、接收键盘输入read

read [选项][变量名]
-p “提示信息”
-t 秒数 #等待时间
-n 指定接受字符数
-s #隐藏输入的数据
read用法举例

十一、declare声明变量类型

declare [+/-][选项] 变量名
选项
- 给变量设定类型属性
+ 取消变量的类型属性
-a 将变量声明为数组型
-i 将变量声明为整数型 declare –i c=$a+$b
-x 将变量声明为环境变量 declare –x test=123和export作用相似，但其实是declare命令的作用
-r 将变量声明为只读变量
-p 显示指定变量的被声明的类型，没有指定变量则查询所有变量的属性
