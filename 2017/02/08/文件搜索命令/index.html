文件搜索命令

发表于 2017-02-08   |   分类于 基础   |  

linux基础知识，常用文件搜索命令用法及区别

locate

用法：locate 文件名
优点：搜索速度快
缺点：只能按文件名搜索。对于新建的文件，只有更新数据库后才能被检索到，系统自定义是每天检索一次（用户可以使用updatedb命令手动更新数据库）

whereis 搜索命令所在路径及帮助文档所在位置

用法：whereis 命令名

which 搜索命令所在路径及别名

用法：which 命令名
whereis跟which找不到shell自带的命令，如cd

find 搜索文件

用法：find 搜索范围 搜索条件 文件名

find / -name in[ab].c

find命令搜的是文件名完全一致的文件,如果想模糊匹配,需要用通配符
* 匹配0或多个字符
? 匹配任意一个字符
[list] 匹配list中的任意单一字符
[!list] 匹配除list中的任意单一字符以外的字符
[c1-c2] 匹配c1-c2中的任意单一字符,如：[0-9] [a-z]
{string1,string2,...}匹配sring1或string2(或更多)其一字符串
{c1..c2} 匹配c1-c2中全部字符,如{1..10}

搜索条件：

  1. -name –i（文件名忽略大小写）
  2. –atime（文件访问时间）
  3. -ctime（文件属性改变时间）
  4. -mtime（文件内容修改时间）
    • find /var/log/ -mtime +10 #查找10天前修改的文件
  5. -size（文件大小）
    • find /etc/ -size +20k –a –size -50k
    • 查找/etc/目录下大于20kb并且小于50kb的文件（-a是and逻辑与，-o是-or逻辑或，+是大于，-是小于，没有符号是等于）
  6. -inum（i节点）
  7. –exec 命令 {}\; （执行查找到的文件）
    • find /root –inum 262421 –exec rm –rf {}\;

grep 在文件中匹配符合条件的字符串

格式：grep [选项] 字符串 文件名
-i：忽略大小写
-v：排除指定字符串

find命令与grep命令的区别

  • find命令：在系统中搜索符合条件的文件名，如果需要匹配，使用通配符匹配，通配符是完全匹配
  • grep命令：在文件当中搜索符合条件的字符串，如果需要匹配，使用正则表达式进行匹配，为包含匹配
